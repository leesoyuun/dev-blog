# garbage-collection 스킬

## 실행 순서

### 1. _workspace/ 정리

- 30일 이상 경과 산출물 삭제
- 현재 마일스톤과 무관한 오래된 파일 삭제

### 2. 코드 품질 점검

`code-reviewer`를 사용해 변경된 파일 전체 스캔:
- 규칙 위반 패턴 집계
- 3회 이상 반복 위반 → 규칙 파일 강화 제안

### 3. 하네스 자가진화

반복 위반 패턴 발견 시:
1. 위반 패턴과 권장 fix를 정리
2. 사용자에게 규칙 파일 업데이트 제안 (승인 필수)

### 4. 문서 동기화 점검

- `docs/TASKS.md`가 실제 완료 상태 반영하는지 확인
- CLAUDE.md의 스택/아키텍처 기술이 현재 코드와 일치하는지 확인

## 출력

```
GC 완료 리포트

_workspace/ 정리: {n}개 파일 삭제
코드 품질: {n}개 위반 발견
  - 반복 위반 (3회+): {패턴}
하네스 업데이트 제안: {있음|없음}
문서 동기화: {동기|불일치}
```
