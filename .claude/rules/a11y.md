# Accessibility (a11y)

## 기본 원칙

WCAG 2.1 AA 기준. 모든 인터랙티브 요소는 키보드로 접근 가능해야 한다.

## 필수 규칙

### 시맨틱 HTML

```tsx
// ✅ 의미있는 태그
<main>, <nav>, <article>, <section>, <aside>, <header>, <footer>

// ❌ div soup
<div className="nav"> ... </div>
```

### 이미지

```tsx
// ✅ 정보 이미지 — 내용 설명
<Image src="/og.png" alt="포스트 썸네일: React 19 새 기능" />

// ✅ 장식 이미지 — 빈 alt
<Image src="/decoration.svg" alt="" />

// ❌ alt 누락
<Image src="/icon.png" />
```

### 인터랙티브 요소

```tsx
// ✅ 버튼은 <button>
<button onClick={toggle}>메뉴 열기</button>

// ❌ div를 버튼으로
<div onClick={toggle}>메뉴</div>

// ✅ 아이콘 버튼 — aria-label 필수
<button aria-label="다크모드 전환">
  <Moon className="h-4 w-4" />
</button>
```

### 포커스 관리

- `focus-visible` 스타일 제거 금지 (`outline: none` 금지)
- 모달·드로어 열림 시 포커스 이동, 닫힘 시 원복
- Tab 순서는 DOM 순서와 일치

### 컬러 대비

- 일반 텍스트: 4.5:1 이상
- 큰 텍스트(18px+): 3:1 이상
- CSS 변수로 관리하는 `--foreground`, `--muted-foreground` 사용
